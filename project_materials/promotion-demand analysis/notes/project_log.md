# 促销需求分析：支持库存与渠道规划

## 项目概述
这个项目围绕促销期间的销售表现展开分析，重点想回答三个问题：  
第一，不同渠道的销售和交易变化是什么样；第二，不同区域和车型的表现差异在哪里；第三，停产车型在经销商渠道中的生命周期表现如何。

项目底层数据来自 PostgreSQL 数据库。我先从原始业务表中提取销售、客户、产品和经销商相关数据，再用 SQL 做筛选、聚合和字段构造，最后把整理好的分析表导入 Tableau，搭建成 3 个用于业务决策支持的仪表盘页面。

## 业务问题
本项目主要围绕以下问题展开：

- 促销期间整体销售表现如何变化？
- 经销商渠道和线上渠道的表现是否存在明显差异？
- 销售额主要集中在哪个价格层级？
- 不同车型在不同州的表现差异有多大？
- 停产车型在经销商渠道中的累计销售轨迹有什么不同？

## 数据来源
项目使用了 PostgreSQL 数据库中的几张核心业务表：

- `sales`：交易级销售记录
- `customers`：客户所在城市和州
- `products`：产品、车型和年份信息
- `dealerships`：经销商覆盖信息

在此基础上，我整理出了 3 张分析表，分别对应不同的分析主题。

## 数据提取与清洗
整个项目没有直接把原始表拖进 Tableau，而是先做了数据整理，主要包括：

- 关联销售、客户、产品和经销商表
- 按分析目标筛选特定时间范围
- 过滤不满足阈值的交易记录
- 构造 `price_category`、`transaction_year` 等分析字段
- 计算停产车型的累计销售额
- 生成适合可视化使用的分析表

最终导出的分析表包括：

### 1. `city_sales_analysis`
用于分析不同年份、渠道、价格层级和产品的销售表现。

主要字段：
- `product_id`
- `channel`
- `sales_amount`
- `price_category`
- `transaction_year`
- `sales_transaction_date`

### 2. `cumulative_sales_dealership`
用于分析停产车型在经销商渠道中的累计销售趋势。

主要字段：
- `model`
- `model_year`
- `product_type`
- `sales_transaction_date`
- `total_daily_sales`
- `cumulative_sales`
- `model_and_year`

### 3. `state_model_sales`
用于分析促销期间不同州和不同车型的销售表现。

主要字段：
- `state`
- `model`
- `total_sales`

## Tableau 仪表盘展示

### 仪表盘 1：整体销售概览
![整体销售概览](visuals/dashboard_1_overview.png)

这一页主要从总量、数量和销售结构三个角度看促销期间的整体表现。  
我在这一页放了总销售额、总交易数和平均销售额 3 个 KPI，以及渠道销售额趋势、渠道交易笔数趋势和价格层级销售额对比图。

#### 主要发现
- 2018 年销售额和交易笔数都达到阶段高点，2019 年两个渠道同步回落。
- 销售额变化不只是客单价问题，交易量本身也出现了下降。
- 收入主要集中在 Premium 价格层级，高价值产品是这一分析窗口中的主要收入来源。

---

### 仪表盘 2：区域与车型表现分析
![区域与车型表现分析](visuals/dashboard_2_region_model.png)

这一页主要看区域分布和车型差异。  
我分别展示了 Lemon 车型销售额最高的前 5 个州、Model Chi 销售额最低的后 5 个州，以及各州销售额总览。

#### 主要发现
- Lemon 的销售额集中在少数高表现州，区域分布并不均衡。
- Model Chi 在部分州的表现明显偏弱，说明不同车型的区域适配度存在差异。
- 从各州销售额总览来看，整体呈现头部州集中、尾部州分散的结构。

---

### 仪表盘 3：产品生命周期趋势分析
![产品生命周期趋势分析](visuals/dashboard_3_lifecycle.png)

这一页主要分析停产车型在经销商渠道中的累计销售轨迹。  
我把累计销售趋势图和最终累计销售额对比图放在一起，一张看过程，一张看结果。

#### 主要发现
- 不同停产车型的累计销售轨迹差异明显。
- Model Chi 2014 和 Model Sigma 2015 的累计增长更快，最终累计销售额也更高。
- Blade 2014 生命周期较短，最终累计销售规模有限。
